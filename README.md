# 📅 Google Calendar Auto-Renumberer

Googleカレンダー上の特定パターンのイベント（定例会議、タスク管理、定期イベントなど）に付与された連番を、過去の実績を基準（アンカー）として自動的に再計算・再採番する Google Apps Script (GAS) ツールです。

間に新しい予定を挿入したり、予定を削除して欠番が生じたりした場合でも、スクリプトを実行するだけで全体が正しい連番に即座に修復されます。

## ✨ Features (主な特徴)

- ⚓ **アンカー基準アーキテクチャ**
  未来の予定だけでなく「過去の確定済みイベント」を起点（アンカー）として再計算します。これにより、未来のスケジュールにダミーの数字や一時的な予定が混入していても、過去からの正確なカウントアップによってすべて正しい連番に自動補正されます。
- 🛡️ **堅牢な防衛的設計とAPI最適化**
  差分検知ロジックを実装し、カレンダーのタイトルに「変更が生じる場合のみ」更新APIを発行します。無駄なAPIコールを排除し（N+1問題の回避）、クラウドリソースとクオータ消費を最小限に抑える設計です。
- ⚙️ **柔軟な正規表現サポート**
  ハードコードされた特定のフォーマットに依存しません。正規表現（Regex）をカスタマイズすることで、あらゆる言語・フォーマットの連番管理に適用可能な完全な汎用ツールです。
- 🔒 **セキュアなシークレット管理**
  カレンダーID等の機密情報はコード内に一切記述せず、GASのスクリプトプロパティ（環境変数）から動的に読み込むセキュアな設計を採用しています。

## 🚀 Setup (導入手順)

本スクリプトは、セキュリティ保護のためカレンダーIDをコード内に直書きしません。以下の手順に従って環境変数をセットアップしてください。

1. **Google Apps Script プロジェクトの作成**
   - Google Drive等から新しいGASプロジェクトを作成し、リポジトリの `Code.gs` を貼り付けます。用途に合わせてコード先頭の `CONFIG` オブジェクトを編集してください。
2. **カレンダーIDの取得**
   - 対象となるGoogleカレンダーの設定画面を開き、「カレンダーの統合」セクションにある「カレンダーID」（例: `xxx@group.calendar.google.com`）をコピーします。
3. **スクリプトプロパティ（環境変数）の設定**
   - GASエディタの左側メニューから「プロジェクトの設定（歯車アイコン）」を開きます。
   - 画面下部の「スクリプト プロパティ」で「スクリプト プロパティを追加」をクリックし、以下を設定して保存します。
     - **プロパティ**: `CALENDAR_ID`
     - **値**: `コピーしたカレンダーID`
4. **実行と権限承認**
   - エディタに戻り、`renumberEvents` 関数を実行します。初回のみGoogleの認可プロセス画面が表示されるため、画面の指示に従いアクセスを許可してください。

## 🛠️ Configuration Examples (設定のカスタマイズ例)

用途に合わせてコード内の `CONFIG` オブジェクトを書き換えることで、様々なフォーマットに対応可能です。

### ケース1: シンプルなプレフィックス （例: `Vol.001 企画会議`）
最も単純な「接頭辞＋数字」のパターンです。
```javascript
const CONFIG = {
  MATCH_REGEX: /^Vol\.(\d+)/, // 先頭が「Vol.」で始まり、続く数字を抽出
  NUMBER_GROUP_INDEX: 1,      // (\d+) は1番目のカッコ
  REPLACE_FORMAT: 'Vol.{{NUMBER}}',
  NUMBER_PADDING: 3,          // 3桁ゼロ埋め
  // ...略...
};
```

### ケース2: 前後に文字があるパターン （例: `【第01回】定例MTG`）
数字の前後に決まった文字があるパターンです。
```javascript
const CONFIG = {
  MATCH_REGEX: /^【第(\d+)回】/, // 先頭が「【第〇〇回】」で始まるものを抽出
  NUMBER_GROUP_INDEX: 1,        // (\d+) は1番目のカッコ
  REPLACE_FORMAT: '【第{{NUMBER}}回】',
  NUMBER_PADDING: 2,            // 2桁ゼロ埋め
  // ...略...
};
```

### ケース3: 動的な状態マーカーを含む複雑なパターン （例: `[Task-001_A] 〇〇実装`）
数字の後ろに都度変わるアルファベット等が含まれており、そのマーカーを保持したまま数字部分だけを連番にしたい場合の高度な設定です。
```javascript
const CONFIG = {
  MATCH_REGEX: /^\[Task-(\d+)_([A-Z])\]/, // 1:数字, 2:任意のアルファベット大文字
  NUMBER_GROUP_INDEX: 1,                  // 数字は1番目のカッコ (\d+)
  // $2 を指定することで、元のタイトルにあった「A」等のマーカーをそのまま引き継ぎます
  REPLACE_FORMAT: '[Task-{{NUMBER}}_$2]', 
  NUMBER_PADDING: 3,
  // ...略...
};
```
